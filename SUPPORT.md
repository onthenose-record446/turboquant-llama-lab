# Support

## Getting Help

Use the repository issues for:

- bug reports
- reproducible benchmark regressions
- documentation gaps
- feature requests

Use the documentation first:

- [docs/QUICKSTART.md](docs/QUICKSTART.md)
- [docs/USAGE_PROFILES.md](docs/USAGE_PROFILES.md)
- [docs/VALIDATED_MODELS.md](docs/VALIDATED_MODELS.md)
- [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)

## Good Support Requests

The most helpful requests include:

- your model name and quantization
- your GPU and driver context
- exact command used
- expected vs actual behavior
- logs or benchmark output

## What This Project Does Not Promise

- universal speed wins on every backend
- paper-level results on all hardware
- turnkey support for every model family without validation
